# PoT SDKs

**Software Development Kits for PoT Microservices**

The PoT SDKs provide libraries for developing microservices that can communicate with each other within an Edge Compute Network without needing to handle the complex discovery process yourself.

## Available SDKs

### Go SDK

The Go SDK for PoT provides an easy way for your edge microservices to communicate with each other.

**Repository**: [https://github.com/Datasance/iofog-go-sdk](https://github.com/Datasance/iofog-go-sdk)

**Documentation**: [Go SDK Documentation](sdk/golang/)

### Other Languages

While the Go SDK is maintained by Datasance, other language SDKs are available from the Eclipse ioFog community:

- **C#**: [eclipse-iofog/iofog-csharp-sdk](https://github.com/eclipse-iofog/iofog-csharp-sdk)
- **C/C++**: [eclipse-iofog/iofog-c-sdk](https://github.com/eclipse-iofog/iofog-c-sdk)
- **Java**: [eclipse-iofog/iofog-java-sdk](https://github.com/eclipse-iofog/iofog-java-sdk)
- **JavaScript (Node.js)**: [Datasance/iofog-nodejs-sdk](https://github.com/Datasance/iofog-nodejs-sdk)
- **Python**: [eclipse-iofog/iofog-python-sdk](https://github.com/eclipse-iofog/iofog-python-sdk)

## Using the SDKs

The SDKs use the [Agent Local API](https://docs.datasance.com/reference-agent/local-api) under the hood, which is a public API you can also call directly if needed.
